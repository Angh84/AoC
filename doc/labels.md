# GitHub Labels

Standardized label set used across all personal repos.

## Type — what kind of work

| Label | Color | Description |
|-------|-------|-------------|
| `bug` | #d73a4a | Something's broken |
| `feature` | #a2eeef | New functionality |
| `chore` | #fef2c0 | Refactoring, deps, tooling, cleanup |
| `docs` | #0075ca | Documentation |

## Size — how big

| Label | Color | Description |
|-------|-------|-------------|
| `size:small` | #c5def5 | Quick win, under an hour |
| `size:medium` | #bfd4f2 | A session or two |
| `size:large` | #0e8a16 | Multi-session effort |

## Horizon — when to tackle it

| Label | Color | Description |
|-------|-------|-------------|
| `near-term` | #0E8A16 | Current focus |
| `medium-term` | #FBCA04 | Next up |
| `long-term` | #1D76DB | Someday/maybe |

## Usage

Every issue gets exactly one **type** label. **Size** and **horizon** are optional but recommended for backlog grooming.
